Design Decisions:
    For the majority of the class interfaces, I chose to use those described in the slides, trusting it would prove advantageous in future assignments and a satisfactory final product.
    For the base classes, data access is done through class methods, instead of directly referencing the data (ie vector.x() instead of vector.x ), although I did leave most data public so as not to lead to frustration in the future. I like the method interface generally because then you can create a more general feel throughout the class interfaces and provides another opportune place to debug. The dot and cross product (and the like) functions are independent functions as opposed to class functions (ie dot(v1,v2) vs. v1.dot(v2) ).
    This type of design seems to have worked well so far. Only a couple of things have come back to bite me.
